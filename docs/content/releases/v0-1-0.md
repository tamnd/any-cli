The first tagged release of `any`. It carries two things: the scaffolder that
writes a complete `tamnd/*-cli` repository, and the kit framework that those
repositories build on.

## The scaffolder

`any new <site>` renders a new `<site>-cli` repository from one proven template:

```bash
any new reddit            # scaffold ./reddit-cli
any new reddit --remote   # also create tamnd/reddit-cli and push
```

A scaffolded repo starts complete:

- A **command tree** (`cli`) with `version`, ready for your subcommands.
- A **library package** with a paced, retrying HTTP client and room for your
  data models.
- **GoReleaser**: archives, deb/rpm/apk, a multi-arch GHCR image, checksums,
  cosign signatures, SBOMs, and Homebrew and Scoop entries that self-disable
  until their tokens exist.
- The **ci, release, and docs** GitHub Actions.
- A **tago docs site** that deploys to GitHub Pages and Cloudflare.

## The kit framework

This release introduces `kit`, the library a scaffolded CLI builds on. A domain
declares an operation once and kit exposes it as three surfaces: a CLI
subcommand, an HTTP route, and an MCP tool.

An operation is a typed handler with surface-neutral metadata. kit reflects the
input struct to learn its arguments and flags, and the output struct to learn
the record's collection name and primary key:

```go
type searchIn struct {
	Query  string        `kit:"arg" help:"URL or prefix to search"`
	Limit  int           `kit:"flag" default:"20" help:"max captures"`
	Client *crawl.Client `kit:"inject"`
}

kit.Handle(app, kit.OpMeta{
	Name:    "search",
	Summary: "Find captures for a URL or prefix",
}, func(ctx context.Context, in searchIn, emit func(Capture) error) error {
	return in.Client.Search(ctx, in.Query, in.Limit, emit)
})
```

That one registration answers `mybin search`, `GET /search` after `mybin serve`,
and a `search` MCP tool after `mybin mcp`, all rendering the same record stream.

Highlights in this release:

- **One registration, four surfaces.** `kit.Handle` builds the CLI, HTTP, and
  MCP surfaces from a single typed handler.
- **A cobra-free escape hatch.** Verbs that do not fit the emit-records shape (a
  byte stream, a shell, a bulk download, a grouping parent) declare a
  `kit.Command` with a `kit.FlagSet` and the `kit.ExactArgs` argument checks, so
  a domain stays free of a direct cobra or pflag import.
- **Typed client access.** `kit.MustClient[T]` and `kit.Client[T]` hand a
  command its domain client off the context.
- **Stable results everywhere.** The `kit/errs` taxonomy maps each failure to a
  fixed CLI exit code and HTTP status, so the same error behaves the same on
  every surface.
- **An optional record store.** A read run with `--db` tees its records into a
  SQLite store as it renders them.

See the [kit README](https://github.com/tamnd/any-cli/blob/main/kit/README.md)
for the full guide.

## Install

```bash
go install github.com/tamnd/any-cli/cmd/any@latest
```

Or take a prebuilt binary from the
[release page](https://github.com/tamnd/any-cli/releases/tag/v0.1.0). See
[installation](/getting-started/installation/) for every option.
